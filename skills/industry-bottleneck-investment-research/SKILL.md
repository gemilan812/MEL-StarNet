# 产业链瓶颈与预期差选股研究 Skill

## Description

本 Skill 用于从产业趋势、供需错配、产业链瓶颈、财报验证、资金扩散和催化剂中提炼研究型选股框架。

本 Skill 不预测股价，不推荐买入或卖出，不承诺收益，只输出可验证、可迁移、可证伪的投资研究分析。

本 Skill 可用于半导体、AI、算力、机器人、新能源、金属、化工、消费、医药、军工、游戏、白酒等不同产业，也可用于 A 股、港股、美股、ETF、日韩台市场和全球产业链映射。

所有输出仅供研究使用，不构成投资建议。

## When to Use This Skill

当用户需要以下分析时使用本 Skill：

1. 发现一个产业主题，想寻找核心受益环节和研究标的。
2. 看到上游材料、零部件、设备或服务涨价，想分析潜在受益公司。
3. 看到龙头公司财报超预期，想寻找产业链扩散机会。
4. 想分析 A 股、港股、美股、ETF 或海外区域市场中的主题投资机会。
5. 想从新闻、财报、公告、研报、推文、会议纪要或产业数据中提炼交易逻辑。
6. 想判断某个概念是否已经被市场充分定价。
7. 想研究资金是否从龙头扩散到供应链侧翼、小市值标的或区域市场。
8. 想识别行业中的瓶颈环节、预期差标的、催化剂和逻辑证伪条件。

## Inputs

尽量要求用户提供：

1. 产业主题或事件，例如 AI 算力、CPO、机器人、固态电池、液冷、白酒复苏、创新药出海。
2. 已知核心标的，例如行业龙头、上游供应商、中游制造商、下游客户、ETF 或海外映射标的。
3. 上游、中游、下游产业链信息，包括原材料、核心设备、关键零部件、制造、封装/集成、渠道、客户、终端应用。
4. 财报、公告、新闻、研报、推文或会议纪要，包括收入、利润、毛利率、订单、产能、价格、库存、客户、认证、资本开支。
5. 当前市场表现，包括股价表现、成交量、板块轮动、ETF 资金流、期权波动率、机构覆盖变化。
6. 关注市场范围：A 股、港股、美股、日韩台市场、全球市场、ETF、产业链映射。
7. 用户风险偏好：稳健型、平衡型、高弹性型、只做研究、不涉及交易。
8. 用户希望输出的格式：产业链拆解表、标的池、研究报告、交易假设、风险清单、催化剂时间线、复盘框架。

如果关键信息缺失，必须明确写：`需要进一步验证`。不得补编数据。

## Core Philosophy

1. 不只看热门概念，而是寻找产业链中最紧缺、最有弹性、最容易被资金重新定价的环节。
2. 不只看龙头，而是寻找龙头需求外溢后，产业链中尚未被充分定价的受益者。
3. 不只看新闻，而是判断新闻是否会改变收入、利润、订单、价格、产能利用率、毛利率或市场预期。
4. 不只看个股逻辑，而是看产业链、资金流、预期差、催化剂和财报验证是否共振。
5. 好机会通常来自：产业趋势确定 + 供需错配 + 市场认知不足 + 催化剂临近。
6. 瓶颈不只来自产能不足，也可能来自技术架构、客户认证、产能分配、供应商稀缺和成品输出控制权。
7. 资金不会永远停留在最明显的龙头上。主线被验证后，资金可能向上游、侧翼、小市值、海外映射和区域市场扩散。
8. 所有逻辑都必须设置证伪条件。没有证伪条件的逻辑只能视为叙事，不能视为研究结论。

## Evidence Discipline

每次分析必须区分：

1. **明确事实**：用户材料、公告、财报、新闻或推文中明确提到的信息。
2. **归纳逻辑**：基于事实推导出的产业链、供需或交易逻辑。
3. **研究框架**：从案例中提炼出的可复用方法。
4. **风险与证伪条件**：哪些情况会削弱或推翻该逻辑。

约束：

- 不得虚构数据、订单、客户、收入占比、市场份额、估值、政策或业绩。
- 如果数据来自推文、论坛或非正式渠道，必须标注：`来源需要核验`。
- 如果只提供单一信息源，必须提示信息偏差和幸存者偏差风险。
- 涉及财报、资金流、估值、客户关系和订单时，必须提醒用户查验原始来源。
- 不得把历史涨幅、博主成功案例或单次上涨结果当成未来依据。

## Workflow

### Step 1: 识别主线事件

先判断事件属于哪一类：

| 事件类型 | 说明 |
|---|---|
| 技术突破 | 新技术路线出现，可能改变成本、效率、性能或产业格局 |
| 产业扩产 | 龙头或下游客户增加资本开支，拉动供应链需求 |
| 产品涨价 | 上游材料、零部件、设备或服务价格上涨 |
| 供给短缺 | 产能不足、原料受限、认证周期长、扩产慢 |
| 财报超预期 | 龙头或供应链公司收入、利润、订单、毛利率超预期 |
| 政策驱动 | 政策补贴、产业规划、进口替代、监管变化 |
| 龙头订单增长 | 核心客户订单增长，带动配套供应链 |
| 新应用爆发 | 新需求场景出现，例如 AI、机器人、智能汽车、低空经济 |
| 海外映射 | 海外产业趋势映射到 A 股、港股或其他市场 |
| 期权/资金异动 | ETF 流入、期权波动率上升、机构增配、空头回补 |
| 库存周期反转 | 行业从去库存进入补库存或涨价周期 |

输出时说明：

- 事件属于哪一类。
- 事件影响需求、供给、价格、订单、利润率还是估值预期。
- 事件是否已被市场充分讨论。
- 事件是否有可验证数据。

### Step 2: 拆解产业链

按以下层级拆解：

| 层级 | 分析重点 |
|---|---|
| 原材料 | 是否稀缺、是否涨价、是否受资源或产能限制 |
| 核心设备 | 是否扩产必需、是否国产替代、是否交付周期长 |
| 关键零部件 | 是否影响整机性能、是否认证周期长、是否供应商少 |
| 制造环节 | 是否产能紧张、良率是否关键、产能利用率是否提升 |
| 封装/集成 | 是否成为新技术路线的关键环节 |
| 渠道/客户 | 是否绑定核心客户，客户需求是否真实增长 |
| 终端应用 | 是否处于需求爆发期，是否具备持续性 |
| 替代技术 | 是否存在被新路线替代的风险 |
| 配套服务 | 测试、认证、运维、软件、能源、电力等支撑环节 |

必须回答：

1. 哪个环节最紧缺？
2. 哪个环节议价能力最强？
3. 哪个环节利润弹性最大？
4. 哪个环节市场关注度最低？
5. 哪个环节最可能被资金从龙头扩散过去？
6. 哪个环节看似普通，但实际存在客户认证、技术架构、产能分配或输出控制权壁垒？

### Step 3: 寻找瓶颈环节

从以下角度判断瓶颈：

- 产能是否有限。
- 扩产周期是否长。
- 技术壁垒是否高。
- 替代供应商是否少。
- 认证周期是否长。
- 客户绑定是否深。
- 毛利率是否可能上行。
- 涨价是否已经出现。
- 库存是否处于低位。
- 下游需求是否快速增长。
- 产能分配是否锁定。
- 公司是否控制成品输出或交付节奏。

瓶颈分类：

| 类型 | 含义 |
|---|---|
| 硬瓶颈 | 供给受限、替代困难、扩产周期长 |
| 软瓶颈 | 阶段性供需错配，后续可能随扩产缓解 |
| 资源型瓶颈 | 原材料稀缺或供给集中 |
| 设备型瓶颈 | 关键设备交付慢、技术门槛高 |
| 工艺型瓶颈 | 良率、制造能力、制程能力限制供给 |
| 认证型瓶颈 | 客户验证周期长，短期难以切换供应商 |
| 架构型瓶颈 | 产品在特定技术路线中不可简单替代 |
| 产能分配型瓶颈 | 未来产能 allocation 被锁定，其他客户争夺有限供给 |
| 输出控制型瓶颈 | 公司控制成品交付或供应节奏，因此可能拥有议价能力 |
| 概念性瓶颈 | 有叙事但缺少数据验证 |

### Step 4: 建立标的池

对每个潜在标的分类：

| 类型 | 含义 |
|---|---|
| 直接龙头 | 主题中最明确受益的公司 |
| 上游瓶颈 | 原材料、设备、零部件中的短缺环节 |
| 中游弹性 | 订单或产能利用率可能快速提升的公司 |
| 下游放量 | 终端需求爆发的公司 |
| 海外映射 | 海外产业趋势映射到本土市场的标的 |
| 预期差标的 | 市场尚未充分认识到受益逻辑的公司 |
| 补涨扩散 | 龙头大涨后资金可能扩散到的相关公司 |
| 观察名单 | 逻辑存在但证据不足，需要继续验证的公司 |

对每个标的检查：

1. 公司是否真的参与该产业链？
2. 相关业务收入占比是否足够？
3. 是否拥有核心客户、核心技术、核心产能或核心认证？
4. 是否处于瓶颈环节，而不是只和概念弱相关？
5. 当前财报是否已经体现该逻辑？
6. 市场是否已经充分定价？
7. 是否存在明确催化剂？
8. 是否存在明确证伪条件？
9. 是否存在流动性不足、估值过高或情绪过热风险？

相关业务占比低、证据不足或逻辑无法验证的公司，只能放入观察名单，并标注 `概念性较强` 或 `需要进一步验证`。

### Step 5: 判断预期差

从以下问题判断：

1. 市场是否只关注龙头，忽略上游或侧翼公司？
2. 公司业务是否和热门主题有关，但市场尚未充分认知？
3. 财报是否尚未体现新订单或涨价影响？
4. 产能、价格、订单是否正在变化，但股价尚未反映？
5. 机构覆盖是否不足？
6. 市值是否较小，业绩弹性是否更大？
7. 产业逻辑是否已经被数据验证？
8. 市场是否使用了错误估值指标，例如只看 TTM 收入而忽略明确量产节点？
9. 市场是否忽略认证周期、技术架构或客户绑定？
10. 市场是否把“同类产品”误判为“可完全替代产品”？

预期差来源：

| 类型 | 说明 |
|---|---|
| 认知预期差 | 市场不知道公司与主题有关 |
| 产业链预期差 | 市场只看龙头，忽略上游瓶颈 |
| 财务预期差 | 当前财报尚未体现未来订单或价格变化 |
| 技术预期差 | 市场误解产品差异或替代难度 |
| 产能预期差 | 市场低估产能分配权或交付能力 |
| 估值预期差 | 市场用 TTM 收入或历史估值，忽略未来放量 |
| 区域预期差 | 资金开始流入某一国家或市场，但部分投资者尚未配置 |
| 资金预期差 | ETF、期权、外资、机构仓位变化尚未被个股充分反映 |

### Step 6: 寻找催化剂

列出可能推动重估的催化剂：

| 催化剂 | 说明 |
|---|---|
| 财报发布 | 收入、利润、毛利率、订单、指引变化 |
| 业绩预告 | 提前确认业绩趋势 |
| 产品涨价 | 材料、零部件、设备或终端价格上涨 |
| 订单公告 | 大客户、新订单、长期协议、框架合同 |
| 产能投放 | 新产线、新工厂、新设备交付 |
| 政策发布 | 补贴、国产替代、行业规划、监管变化 |
| 龙头财报 | 龙头资本开支、库存、订单、供应链指引 |
| 行业会议 | 新产品、新技术路线、客户合作披露 |
| 新产品发布 | 改变需求结构或供应链格局 |
| 海外龙头上涨 | 海外映射到本土供应链或同类公司 |
| 期权波动率变化 | 市场对未来波动预期上升 |
| 资金流入主题 ETF | 资金进入板块或区域市场 |
| 分析师覆盖增加 | 研究覆盖提高，市场认知扩散 |
| 资格认证进展 | 客户认证通过，进入放量前阶段 |
| 并购或纵向整合 | 公司扩大 TAM 或增强产业链控制力 |

每个催化剂都必须配套“需要验证的数据”。不得把远期量产直接等同于业绩兑现。

### Step 7: 判断交易阶段

| 阶段 | 特征 | 研究动作 |
|---|---|---|
| 认知萌芽期 | 少数人讨论，股价反应弱，产业数据刚出现变化 | 深入研究基本面，建立初始 thesis |
| 逻辑扩散期 | 产业链相关标的开始异动，市场开始讨论供应链受益者 | 建立标的池，区分核心与边缘标的 |
| 数据验证期 | 财报、订单、涨价、产能、客户验证逻辑 | 筛选强兑现公司，剔除概念性标的 |
| 资金加速期 | 板块普涨，龙头明显走强，ETF 或外资流入 | 检查估值、拥挤度和交易节奏 |
| 情绪高潮期 | 叙事过热，弱相关标的也上涨 | 控制风险，重新检查证伪条件 |
| 逻辑证伪期 | 业绩不兑现、价格下跌、供需逆转、客户流失 | 退出研究池或降低研究权重 |

研究到交易的映射流程：

```text
初始研究
-> 产业链拆解
-> 找到瓶颈环节
-> 映射标的池
-> 判断预期差
-> 跟踪催化剂
-> 等待财报、订单、价格、资金流验证
-> 根据验证结果升级、降级或剔除标的
```

不得仅因主题上涨就默认仍有吸引力，必须重新检查估值、拥挤度和基本面验证程度。

### Step 8: 风险检查

必须检查：

1. 逻辑是否只是概念炒作？
2. 公司收入中相关业务占比是否太低？
3. 涨价是否可持续？
4. 供给是否很快释放？
5. 下游需求是否真实？
6. 财报是否能验证逻辑？
7. 股价是否已经充分反映？
8. 估值是否过高？
9. 是否存在监管、政策或地缘风险？
10. 是否存在流动性和情绪退潮风险？
11. 客户是否会推动多源化，削弱单一供应商议价权？
12. 技术路线是否可能切换？
13. 量产时间是否过远，导致市场无法持续等待？
14. 是否存在幸存者偏差，只看到成功案例，忽略失败案例？
15. 数据来源是否可靠，是否需要进一步核验？

## Generalization Rules

1. 先找产业趋势，再找受益环节。
2. 先找供需错配，再找利润弹性。
3. 先找瓶颈环节，再找具体标的。
4. 先判断市场认知，再判断预期差。
5. 先找催化剂，再判断交易节奏。
6. 先验证基本面，再考虑价格弹性。
7. 任何个股都不能只因为“概念相关”就列为核心标的。
8. 必须说明逻辑证伪条件。
9. 资金流只能作为辅助验证，不能替代基本面验证。
10. 不得把瓶颈永久化，必须检查产能、技术、认证、客户和替代供应商变化。
11. 不能把龙头上涨直接等同于全产业链上涨。
12. 不能把小市值等同于高弹性，必须同时验证业务相关性和兑现能力。
13. 不能只看收入规模，还要看毛利率、订单、产能、客户、认证和价格变化。
14. 远期逻辑如果缺少中短期验证，必须提示估值回撤风险。

## Constraints

1. 不推荐买入或卖出。
2. 不承诺收益。
3. 不使用确定性收益表述。
4. 不把历史涨幅当成未来依据。
5. 不虚构数据。
6. 如果缺少数据，必须明确写 `需要进一步验证`。
7. 如果标的相关性弱，必须降级为观察名单。
8. 如果逻辑无法验证，必须标注为 `概念性较强`。
9. 不得将短期资金流入等同于长期基本面改善。
10. 不得将单一案例直接泛化为所有行业规律。
11. 如果用户要求给出确定投资结论，改为输出研究框架和风险条件。
12. 如果用户要求预测股价，拒绝确定性预测，只给出影响变量和情景分析。
13. 如果标的已经显著上涨，必须检查是否进入情绪高潮期或估值透支期。
14. 如果行业主线依赖远期放量，必须检查中短期是否有验证节点。

## Reasoning Template

分析新主题时按以下顺序思考：

1. 这是不是一个真实产业趋势？
2. 这个趋势会增加哪个环节的需求？
3. 哪个环节最可能供给不足？
4. 供给不足来自资源、产能、技术、认证、客户绑定、产能分配，还是输出控制？
5. 哪些公司真正控制这个瓶颈？
6. 市场是否已经充分理解这些公司？
7. 当前财报是否已经体现该逻辑？
8. 未来有什么催化剂可以验证？
9. 哪些数据会证伪逻辑？
10. 当前处于认知萌芽期、逻辑扩散期、数据验证期、资金加速期、情绪高潮期，还是逻辑证伪期？

## Data Verification

对任何具体主题，尽量核验财报、价格、产能、客户、认证、资金、估值、替代风险和证伪数据。详细清单见 [verification-checklist.md](references/verification-checklist.md)。

## Preferred Response Style

1. 先给结论，再给分析。
2. 用表格呈现产业链、标的、催化剂和风险。
3. 明确区分事实、推断、框架和需要验证的信息。
4. 不使用夸张、情绪化或确定性收益语言。
5. 对高风险、小市值、远期放量标的必须单独提示风险。
6. 对推文、论坛、社交媒体中的说法必须标注 `来源需要核验`。
7. 对缺少数据的部分，不编造，不脑补，直接写 `暂无足够信息` 或 `需要进一步验证`。
8. 结尾必须给出后续跟踪重点。

## Quality Checklist

输出前检查：

- 是否区分了事实、逻辑、框架和风险。
- 标的是否对应明确产业链位置。
- 每个催化剂是否有可验证数据。
- 每个核心标的是否有清晰分层理由。
- 弱相关标的是否被降级为观察名单。
- 是否检查了估值、拥挤度、资金流反转和基本面证伪。
- 结论是否保持研究属性，而非投资建议。

## Final Reminder

本 Skill 的目标是帮助用户形成产业链研究框架，而不是输出投资指令。

任何标的都必须经过产业链相关性、财报兑现、催化剂、估值、资金流和风险条件的交叉验证。

如果逻辑无法被数据验证，该逻辑只能作为观察线索，不能作为核心研究结论。
