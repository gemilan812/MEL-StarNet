# Data Verification Checklist

对于任何具体主题，尽量核验以下数据。缺少数据时写 `需要进一步验证`。

| 数据类型 | 需要核验的问题 |
|---|---|
| 财报数据 | 收入、利润、毛利率、订单、库存、资本开支是否真实改善 |
| 价格数据 | 产品是否涨价，涨价是否持续，价格来源是否可靠 |
| 产能数据 | 当前产能、扩产计划、投产时间、良率是否明确 |
| 客户数据 | 是否进入核心客户供应链，是否有公告或可靠来源 |
| 认证数据 | 是否通过客户认证，认证周期是否仍在进行 |
| 资金数据 | ETF 流入、外资买入、机构持仓、期权波动率是否变化 |
| 估值数据 | 当前估值是否基于合理假设，是否透支远期增长 |
| 替代风险 | 是否存在其他供应商、其他技术路线或多源化趋势 |
| 证伪数据 | 哪些数据出现后说明逻辑失败 |

## Source Reliability

| 来源 | 使用方式 | 注意事项 |
|---|---|---|
| 公司公告 / 年报 / 季报 | 优先作为事实来源 | 仍需检查口径、一次性因素和管理层指引 |
| 交易所公告 / 监管文件 | 用于核验重大合同、融资、并购、风险提示 | 注意披露时间和适用市场规则 |
| 行业协会 / 海关 / 价格数据库 | 用于价格、库存、产量、进出口验证 | 注意滞后性和样本覆盖 |
| 券商研报 / 专家纪要 | 用于形成假设和交叉验证 | 不能直接当作事实，需核验原始来源 |
| 推文 / 论坛 / 社交媒体 | 用于发现线索 | 必须标注 `来源需要核验`，防止幸存者偏差 |
| 股价 / 成交量 / 期权 / ETF 资金流 | 用于观察交易行为 | 只能辅助验证，不能替代基本面 |

## Disqualification Rules

出现以下情况时，应降级为观察名单或剔除：

1. 公司与主题只有概念关联，缺少收入、客户、订单或产品证据。
2. 相关业务占比过低，且短中期看不到放量路径。
3. 供应瓶颈已被新增产能、替代技术或多供应商策略缓解。
4. 远期量产时间过长，缺少中短期财报、订单、认证或价格验证节点。
5. 股价已经显著反映乐观预期，但新增证据不足。
6. 推文或论坛信息无法被公告、财报、产业数据或第三方来源交叉验证。
7. 逻辑只依赖“龙头上涨后补涨”，缺少基本面传导路径。

## Industry-Agnostic Examples

这些示例仅用于说明框架，不构成投资建议。

### 半导体 / AI 算力

分析路径：

```text
AI CapEx 增长
-> GPU、服务器、交换机需求上升
-> 存储、光模块、CPO、先进封装、电力、电网需求提升
-> 查找供给受限环节
-> 映射材料、设备、测试、封装、区域市场标的
-> 跟踪财报、订单、涨价、资金流
```

重点检查：存储价格、光器件供需、先进封装产能、电网约束、资金是否从龙头扩散到供应链。

### 金属 / 化工

分析路径：

```text
下游需求增长或供给受限
-> 原材料价格上涨
-> 成本曲线和产能分布变化
-> 找到低成本产能、稀缺资源或扩产慢的公司
-> 跟踪价格、库存、开工率、订单和政策
```

重点检查：涨价可持续性、库存位置、新增供给、成本上升是否侵蚀利润。

### 机器人 / 智能制造

分析路径：

```text
机器人需求提升
-> 伺服、电机、减速器、传感器、控制器、软件需求增长
-> 查找精密零部件或核心算法瓶颈
-> 区分核心供应商和概念公司
-> 跟踪订单、客户认证、量产节奏
```

重点检查：是否进入核心客户供应链、是否具备技术壁垒、是否存在量产验证。

### 消费 / 白酒

分析路径：

```text
消费复苏或价格体系改善
-> 龙头渠道库存变化
-> 批价、动销、渠道利润改善
-> 资金从龙头扩散到次高端或区域品牌
-> 跟踪库存、批价、回款和节日动销
```

重点检查：需求是否真实恢复、渠道库存是否健康、涨价是否被消费者接受、财报是否验证动销改善。

### 医药 / 创新药

分析路径：

```text
临床数据或出海授权
-> 管线价值重估
-> 上游 CXO、原料药、设备、检测服务可能受益
-> 判断研发节点、商业化路径和支付能力
-> 跟踪临床数据、审批、授权金额和销售放量
```

重点检查：临床数据、商业化路径、估值是否已反映远期乐观预期、管线失败风险。
